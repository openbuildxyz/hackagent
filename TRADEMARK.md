# HackAgent Trademark Guidelines

The Apache-2.0 license covers the source code in this repository. It does not grant trademark rights.

## What is protected

The HackAgent name, HackAgent logo, official domain names, and official project identity belong to the HackAgent project maintainers.

## Allowed uses

You may:

- Fork and modify the code.
- Use the code for personal, internal, commercial, or SaaS products under Apache-2.0.
- Refer to HackAgent to describe compatibility, origin, or attribution.
- Say that your project is "based on HackAgent" or "forked from HackAgent" when that is accurate.

## Not allowed without permission

You may not:

- Present your fork as the official HackAgent project.
- Use the HackAgent name, logo, or official domain in a way that confuses users about source or endorsement.
- Use HackAgent branding for commercial promotion implying official partnership or sponsorship.
- Register confusingly similar domains, package names, or social accounts that impersonate the official project.

## Naming forks

Use a distinct name for public forks or hosted services. A clear phrase such as "Powered by HackAgent" or "Forked from HackAgent" is acceptable when it does not imply official status.

For trademark permission, contact the maintainers through the security/contact channel listed in `SECURITY.md`.
