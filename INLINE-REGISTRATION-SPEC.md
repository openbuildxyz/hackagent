# HackAgent 报名路径最终调整

## 产品目标

把报名体验从“详情页跳 apply 页再填表”改成“详情页原地完成报名”。

用户看完活动信息后，不需要再经历一次完整页面跳转，就能直接完成报名。

## 结论

主路径必须是：
- 公开活动详情页直接展示报名表
- 点击“立即报名”只做页面内定位/展开，不再把已登录用户送去 `/apply/[eventId]`
- `/apply/[eventId]` 保留为兼容入口，但不再是主入口

## 这次要做的事

### 必做
1. 在公开活动详情页内嵌报名表，直接承载提交动作
2. 复用现有报名字段、校验、提交逻辑，不改后端接口
3. 保留 apply 页，但它应尽量复用同一套报名表组件，避免两份逻辑分叉
4. 已登录且可报名时，详情页 CTA 应直接定位到报名区，而不是跳页

### 不要做
- 不要改报名 API
- 不要改审核状态逻辑
- 不要引入弹窗作为主方案
- 不要把详情页再导向 `/apply/[eventId]` 作为主路径

## 预期交互

- 已登录 + 报名开放：详情页内直接看到报名表
- 点击“立即报名”：滚动到报名区 / 聚焦报名区
- 未登录：引导登录，登录后仍回到详情页原位报名
- 已报名：显示状态卡，不再显示表单
- 报名关闭：显示关闭状态和原因

## 验收标准

- 详情页可以直接完成报名
- 用户从详情页不再被迫进入独立 apply 页
- apply 页仍可访问，且不与详情页逻辑冲突
- typecheck / lint / build 通过
- 生产站点可访问且报名路径符合上述规则
